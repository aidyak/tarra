# 変更ログ（2025-12-27）

## サマリ
- Honoサーバを `api/server.ts` へ移動し、API資源を明確化。
- フロント（React）で `/api/hello` を取得して画面表示するロジックを追加。
- 開発起動スクリプトを `tsx api/server.ts` へ更新。
- Node向けTS設定を `api/**/*.ts` を対象に拡張。

## 変更詳細
- ファイル移動: [api/server.ts](../api/server.ts)
- スクリプト更新: [package.json](../package.json) → `dev`, `dev:api` を `api/server.ts` 指定へ。
- TS設定: [tsconfig.node.json](../tsconfig.node.json) → `include: ["vite.config.ts", "api/**/*.ts"]`。
- フロント実装: [src/App.tsx](../src/App.tsx) → `useEffect`で`fetch('/api/hello')`、表示・ローディング・エラー制御を追加。

## 影響範囲
- 開発者はAPI関連の変更を `api/` 配下で管理。
- フロントは `/api/*` の呼び出しでプロキシを透過利用可能（[vite.config.ts](../vite.config.ts)）。

## 確認手順
```bash
pnpm dev
# ブラウザで http://localhost:5173/ (使用中なら5174等) を開く
# 画面「APIからのメッセージ」に "Hello from Hono" が表示される

# 任意でAPI直アクセス
curl http://localhost:8787/api/hello
```
